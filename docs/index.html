The spec-driven factory for multi-repo systems

A workspace template for Claude Code where every change starts as a spec, humans approve at six permission-backed gates, tests are written first and cannot be edited to pass, and documentation and metrics are derived from the real sources — so they cannot lie.

Architecture & guide (EN) Documento de arquitectura (ES)

📋 Spec-anchored, end to end

Ten phases from triage to operations — not just spec → code. Structured clarification, cross-artifact consistency analysis, code↔spec convergence and blameless postmortems close the loop.

🔒 Gates backed by permissions

Agents hold no production credentials and cannot approve their own work. Every gate records who approved, when, and on which commit — enforced by tool permissions, not by prompt text.

🧭 Built for multi-repo & brownfield

The system topology is data (repos.yaml); an as-is map, a code graph and per-repo context packs with executable assertions keep agents grounded in code that already exists.

📊 Docs and metrics that cannot lie

The architecture document, the as-is map and the DORA metrics are generated from live sources with provenance seals. No source? The cell says "no data" — never a made-up number.

🧪 Strict TDD, verified in CI

Failing tests are committed before implementation and agents are forbidden from editing them. Three self-test suites guard the factory's own tooling; CI fails on documentation drift.

📚 Evidence-based method

The process encodes what DORA, McKinsey, Anthropic and Thoughtworks identify in high performers: small batches, human-in-the-loop, context engineering and system-level metrics.
